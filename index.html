An Introduction to CSS/SASS

Welcome! This page is designed to introduce a few stylish organizational ideas to make working with CSS and SASS a little more awesome. It also is a living reference guide and exercise for proper CSS and SASS techniques. Currently, there are two versions of this page: one filled with numerous CSS anti-patterns, and one that is better organized and conforms to our architecture and style. As an exercise/tutorial, this page is meant to be cleaned-up and tweaked to be more semantic and badass. Careful inspectors of the markup and CSS will notice a few glaring patterns evident in this version. We've got a lack of classes, we're globally changing elements, and ultimately have set up a teetering tower ready to crumble into a million tear-soaked pieces.

The Role of CSS

Most everyone knows that CSS is used for styling content. Any given web page can be separated in to three (not exactly disctinct) categories: (1) Content, (2) Decoration, (3) Interactivity. As you've probably guessed, the content is usually just HTML, the decoration is CSS, and the interactivity is the page's JavaScript. Today we'll talk about a page's "decoration" layer.

Selection

The most fundamental concept in CSS is selection. We've all used CSS selectors before. We've seen that *, the .something or the #hashtag. These commonly used selectors are usually called simple selectors. CSS gets fun (or ugly) when we chain these simple selectors together.

Definition

A CSS combinator combines or explains the relationship between two or more simple selectors.
